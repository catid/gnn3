## Gaps And Risks

- Experiment metadata does not yet record branch name, scout/candidate/contender stage, or explicit device placement string in `summary.json`; continuation runs should add that before longer batches.
- `run_dual_gpu_cycle.sh` does not enforce `CUDA_VISIBLE_DEVICES`; paired runs are safe only if configs pin different devices.
- `OuterHistoryReader` currently reads all prior node states densely. For `B1`, that is the wrong mechanism; continuation should add compressed outer-history summaries so the experiment measures history utility rather than just more dense memory.
- The benchmark generator does not yet expose explicit knobs for hub asymmetry, burstiness, or eval-only OOD sweeps beyond the current queue/capacity settings; `A1` needs additive config knobs rather than ad hoc code edits.
- `A3` needs careful interpretation. The current `outer_steps` is a fixed model construction parameter, so `H_test` studies beyond trained depth require either shared transition reuse or a bounded eval override path.
